## 一、项目概述

### 1.1 项目背景

构建一套金融数据抓取与分析系统，实现从公开数据源（AKShare）自动采集汇率数据，提供数据可视化展示（K线图、均线图、MACD图），并支持数据源配置、采集任务管理等功能。

### 1.2 核心目标

1. **自动化数据采集**：定时从AKShare采集USDCNH汇率数据，入库存储
2. **数据可视化展示**：前端展示K线图、均线图、MACD图
3. **配置化管理**：Web端配置数据源、采集任务参数
4. **用户权限管理**：区分admin/user角色，权限差异化控制

### 1.3 核心原则

| 原则 | 说明 |
|------|------|
| 分段开发 | 第一阶段仅完成技术框架和核心功能，无冗余组件 |
| 规范先行 | 代码注释、日志、配置统一规范 |
| 性能稳定 | 数据库索引、连接池、异步处理、缓存策略保障 |
| AI友好 | 清晰分层、完整示例、规范注释，便于AI辅助开发 |
| 平滑扩展 | 预留后续功能接口，插件式升级 |

## 四、数据流程

### 4.1 数据采集流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  APScheduler │───>│  AKShare     │───>│  PostgreSQL  │
│  定时触发    │    │  数据抓取    │    │  数据入库    │
└──────────────┘    └──────────────┘    └──────────────┘
       │                   │                   │
       │                   │                   │
       v                   v                   v
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Cron配置    │    │  tenacity    │    │  索引优化    │
│  可视化界面  │    │  重试机制    │    │  查询加速    │
└──────────────┘    └──────────────┘    └──────────────┘
```

**流程说明**：

1. **任务调度**：APScheduler根据cron表达式定时触发采集任务
2. **数据抓取**：AKShare采集器调用API获取USDCNH汇率数据
3. **重试机制**：tenacity实现失败重试（最多3次，间隔递增）
4. **数据入库**：SQLAlchemy Async ORM写入PostgreSQL
5. **索引优化**：时间字段、查询字段建立索引加速查询

### 4.2 数据查询流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  前端请求    │───>│  FastAPI     │───>│  PostgreSQL  │
│  时间范围    │    │  API处理     │    │  数据查询    │
└──────────────┘    └──────────────┘    └──────────────┘
       │                   │                   │
       │                   │                   │
       v                   v                   v
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  限制1000条  │    │  内存缓存    │    │  分页查询    │
│  默认30天    │    │  热点数据    │    │  索引命中    │
└──────────────┘    └──────────────┘    └──────────────┘
       │                   │
       │                   │
       v                   v
┌──────────────┐    ┌──────────────┐
│  ECharts     │    │  TA-Lib      │
│  图表渲染    │    │  技术指标    │
└──────────────┘    └──────────────┘
```

**流程说明**：

1. **前端请求**：用户选择时间范围，请求最多1000条数据
2. **缓存检查**：内存缓存检查30天内热点数据
3. **数据查询**：未命中缓存则查询PostgreSQL，利用索引加速
4. **技术指标**：TA-Lib计算MA/MACD指标
5. **图表渲染**：ECharts渲染K线图、均线图、MACD图

### 4.3 用户认证流程

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  用户登录    │───>│  密码验证    │───>│  Session创建 │
│  用户名密码  │    │  bcrypt校验  │    │  PostgreSQL  │
└──────────────┘    └──────────────┘    └──────────────┘
       │                   │                   │
       │                   │                   │
       v                   v                   v
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Cookie返回  │    │  权限检查    │    │  Session存储 │
│  Session ID  │    │  角色判断    │    │  过期时间    │
└──────────────┘    └──────────────┘    └──────────────┘
```

**流程说明**：

1. **登录请求**：用户提交用户名、密码
2. **密码验证**：bcrypt校验密码hash
3. **Session创建**：创建Session记录存入PostgreSQL
4. **Cookie返回**：Cookie携带Session ID返回前端
5. **权限检查**：每次请求检查Session有效性和用户角色
