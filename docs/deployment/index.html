第一章 系统概述

1.1 系统架构

FDAS（金融数据抓取与分析系统）采用六层架构设计，包括基础设施层、数据采集层、数据存储层、后端服务层、前端展示层和集成对接层。

架构图
┌─────────────────────────────────────────────────────────────────┐
│                     第七层：预留扩展层                            │
│  (通用数据源、数据备份、Token监控、性能监控等)                     │
├─────────────────────────────────────────────────────────────────┤
│                     第六层：集成对接层                            │
│                     (飞书 Webhook 告警推送)                       │
├─────────────────────────────────────────────────────────────────┤
│                     第五层：前端展示层                            │
│  Vue 3 + Element Plus + ECharts + Pinia + Vite                  │
├─────────────────────────────────────────────────────────────────┤
│                     第四层：后端服务层                            │
│  FastAPI + SQLAlchemy 2.0 + Pydantic + APScheduler              │
├─────────────────────────────────────────────────────────────────┤
│                     第三层：数据存储层                            │
│                     PostgreSQL 16                                │
├─────────────────────────────────────────────────────────────────┤
│                     第二层：数据采集层                            │
│  Python + AKShare + APScheduler + tenacity + TA-Lib             │
├─────────────────────────────────────────────────────────────────┤
│                     第一层：基础设施层                            │
│               Docker Desktop + Linux容器                         │
└─────────────────────────────────────────────────────────────────┘

1.3 部署前提条件

⚠️ 环境要求：
  • Docker 24+ 和 Docker Compose已安装
  • 至少4GB可用内存
  • 至少20GB可用磁盘空间
  • 网络可访问外部数据源（AKShare）
🔒 安全要求：
  • SESSION_SECRET必须配置（至少32字符）
  • 数据库密码必须修改（非默认值）
  • 首次登录后立即修改admin密码
  • CORS配置为实际域名（非localhost）

1.4 部署方案说明

本指南提供两套部署方案：

方案适用场景特点
单容器部署小规模部署、测试环境、资源受限场景部署简单、资源占用少、单进程管理
多容器部署生产环境、需要独立扩展、高可用场景容器分离、独立扩展、易于维护

推荐生产环境使用多容器部署方案，便于后续扩展和维护。

第二章 单容器部署方案

2.1 方案架构

单容器部署方案将所有服务部署在同一个Linux容器内，使用不同用户隔离服务进程。

架构
单个Linux容器:
├── PostgreSQL（postgres用户，UID 999）
│   └── 数据目录: /var/lib/postgresql/data
│
├── FastAPI后端（fdas-backend用户，UID 1001）
│   ├── 应用目录: /opt/fdas/backend
│   └── 端口: 8001（内部）
│
├── Nginx前端（fdas-frontend用户，UID 1002）
│   ├── 静态文件: /opt/fdas/frontend/dist
│   └── 端口: 8000（外部）
│
└── Supervisor进程管理
    └── 配置: /opt/fdas/config/supervisord.conf

2.2 用户隔离设计

用户UID用途权限范围
postgres999运行PostgreSQL/var/lib/postgresql/*
fdas-backend1001运行FastAPI/opt/fdas/backend/*
fdas-frontend1002前端静态文件/opt/fdas/frontend/dist/*

2.3 一键部署脚本

使用deploy.sh脚本一键完成部署：

bash
# 进入部署目录
cd deployment-packages/single-container

# 创建环境变量文件
cp config/.env.template .env
vim .env  # 编辑配置

# 执行一键部署
./deploy.sh --env-file .env --action deploy

# 验证部署
curl http://localhost:8000/api/health

deploy.sh支持的操作：

操作命令说明
部署--action deploy构建镜像并启动服务
停止--action stop停止所有服务
重启--action restart重启所有服务
日志--action logs查看服务日志
状态--action status查看服务状态

2.4 手动部署步骤

如需手动部署，按以下步骤执行：

Step 1: 安装系统依赖

bash
cd deployment-packages/single-container/scripts
./install-system.sh

Step 2: 创建用户和目录

bash
./setup-users.sh

Step 3: 安装Python依赖

bash
./install-python.sh

Step 4: 初始化数据库

bash
./init-database.sh

Step 5: 启动Supervisor

bash
supervisord -c /opt/fdas/config/supervisord.conf
supervisorctl status

2.5 配置文件说明

环境变量配置 (.env)

bash
# 数据库配置
DB_PASSWORD=your-secure-db-password-here

# Session安全配置（必须32+字符）
SESSION_SECRET=change-this-in-production-min-32-characters

# 调试模式（生产必须false）
DEBUG=false

# CORS配置
ALLOWED_ORIGINS=["https://your-domain.com"]

# 应用端口
APP_PORT=8000

Supervisor配置

Supervisor管理PostgreSQL、FastAPI、Nginx三个进程，配置文件位于config/supervisord.conf。

2.6 应对措施

TA-Lib编译失败：

如遇TA-Lib编译失败，可使用预编译二进制包：

# Debian/Ubuntu
apt-get install -y libta-lib0 libta-lib0-dev

# 或使用conda
conda install -c conda-forge ta-lib
PostgreSQL启动失败：

检查数据目录权限和初始化状态：

# 检查权限
ls -la /var/lib/postgresql/data

# 重新初始化
rm -rf /var/lib/postgresql/data/*
su - postgres -c "initdb -D /var/lib/postgresql/data"

第三章 多容器部署方案

3.1 方案架构

多容器部署方案使用Docker Compose编排前后端容器和数据库容器，实现服务分离。

架构
Docker Compose编排:
├── fdas-db容器
│   ├── 镜像: postgres:16-alpine
│   ├── 端口: 5432
│   ├── 卷: fdas-db-data
│   └── 网络: fdas-network
│
├── fdas-app容器
│   ├── 镜像: 多阶段构建（前端+后端）
│   ├── 端口: 8000
│   ├── 卷: fdas-logs, fdas-data
│   ├── 网络: fdas-network
│   └── 健康检查: /api/health
│
└── Docker网络
    └── fdas-network (172.28.0.0/16)

3.2 Docker Compose配置

yaml
version: '3.8'

services:
  fdas-db:
    image: postgres:16-alpine
    container_name: fdas-db
    environment:
      POSTGRES_USER: fdas
      POSTGRES_PASSWORD: ${DB_PASSWORD:-fdas}
      POSTGRES_DB: fdas
    volumes:
      - fdas-db-data:/var/lib/postgresql/data
      - ../../docker/init-db.sql:/docker-entrypoint-initdb.d/init-db.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U fdas"]
      interval: 10s
      retries: 5

  fdas-app:
    build:
      context: ../../
      dockerfile: deployment-packages/multi-container/Dockerfile.app
    container_name: fdas-app
    environment:
      DATABASE_URL: postgresql+asyncpg://fdas:${DB_PASSWORD}@fdas-db:5432/fdas
      SESSION_SECRET: ${SESSION_SECRET}
    depends_on:
      fdas-db:
        condition: service_healthy
    ports:
      - "8000:8000"
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/api/health"]

volumes:
  fdas-db-data:
  fdas-logs:

networks:
  fdas-network:

3.5 应对措施

容器启动失败：

检查容器日志和健康检查状态：

# 查看容器日志
docker logs fdas-app
docker logs fdas-db

# 检查健康状态
docker inspect fdas-db --format='{{.State.Health.Status}}'

# 重启单个容器
docker-compose restart fdas-app
数据库连接失败：

检查网络和数据库状态：

# 检查网络
docker network inspect fdas-network

# 测试数据库连接
docker exec fdas-app curl fdas-db:5432
docker exec fdas-db pg_isready -U fdas

# 检查数据库日志
docker logs fdas-db | tail -50

第四章 迁移方案

4.1 数据迁移

从单容器迁移到多容器，首先备份数据：

bash
# 进入迁移目录
cd deployment-packages/migration

# 执行数据备份
./migrate-data.sh --source-container fdas-single

# 备份位置
ls -la /tmp/fdas_migration/

数据备份内容包括：

  • 数据库完整备份（PostgreSQL自定义格式）
  • 应用数据卷备份（日志、配置）
  • 环境变量备份

4.2 服务迁移

数据备份完成后，执行服务迁移：

bash
# 执行服务迁移（会自动部署多容器方案）
./migrate-service.sh

# 迁移完成后验证
curl http://localhost:8000/api/health
docker ps

4.3 回滚机制

如迁移失败，可执行回滚恢复到单容器方案：

bash
# 执行回滚
./rollback.sh

# 回滚验证
curl http://localhost:8000/api/health

4.4 迁移验证清单

迁移完成后，请逐项验证：

检查项验证方法预期结果
容器状态docker psfdas-db和fdas-app运行
健康检查curl /api/health返回200
表数量SELECT COUNT(*)大于10
用户数据SELECT COUNT(*) FROM users大于0
登录功能浏览器测试可正常登录

第五章 增量更新

5.1 版本升级流程

使用upgrade.sh脚本执行版本升级：

bash
# 进入更新目录
cd deployment-packages/update

# 模拟升级（不实际执行）
./upgrade.sh --version 2.1.0 --dry-run

# 执行升级
./upgrade.sh --version 2.1.0

# 验证版本
curl http://localhost:8000/api/health

升级流程：

  1. 获取当前版本并创建备份
  2. 检查版本变更日志
  3. 执行数据库迁移（如有）
  4. 拉取新版本代码
  5. 构建新版本镜像
  6. 部署并验证

5.2 数据库迁移模板

数据库迁移脚本模板位于versions/migrations.sql：

sql
-- 版本 X.X.X 数据库迁移脚本

-- 添加新字段
ALTER TABLE forex_symbols ADD COLUMN IF NOT EXISTS decimal_places INTEGER DEFAULT 4;

-- 创建新表
CREATE TABLE IF NOT EXISTS forex_intraday (...);

-- 创建索引
CREATE INDEX IF NOT EXISTS idx_name ON table_name(column);

-- 更新版本记录
INSERT INTO system_info (key, value) VALUES ('db_version', 'X.X.X')
ON CONFLICT (key) DO UPDATE SET value = 'X.X.X';

5.3 回滚机制

如升级失败，执行回滚：

bash
# 回滚到指定版本
./rollback.sh --version 2.0.1

# 验证回滚
curl http://localhost:8000/api/health

附录A 常见问题与应对措施

A.1 TA-Lib编译失败

问题描述：

执行pip install TA-Lib时报编译错误，缺少C库。

应对措施：
  1. 安装TA-Lib C库： 
    wget http://prdownloads.sourceforge.net/ta-lib/ta-lib-0.4.0-src.tar.gz
tar -xzf ta-lib-0.4.0-src.tar.gz
cd ta-lib/
./configure --prefix=/usr
make && make install
ldconfig
  2. 或使用系统包管理器： 
    # Ubuntu/Debian
apt-get install -y libta-lib0 libta-lib0-dev

# CentOS/RHEL
yum install -y ta-lib
  3. 或使用conda： 
    conda install -c conda-forge ta-lib

A.2 数据库连接失败

问题描述：

后端启动时报"connection refused"或"database does not exist"错误。

应对措施：
  1. 检查PostgreSQL状态： 
    # 容器方案
docker exec fdas-db pg_isready -U fdas

# 单容器方案
su - postgres -c "pg_isready"
  2. 检查数据库是否存在： 
    docker exec fdas-db psql -U fdas -l
  3. 初始化数据库： 
    docker exec fdas-db psql -U fdas -d fdas -f /docker-entrypoint-initdb.d/init-db.sql

A.3 环境变量配置错误

问题描述：

启动时报"SESSION_SECRET not configured"或长度不足错误。

应对措施：
  1. 生成安全密钥： 
    python3 -c "import secrets; print(secrets.token_urlsafe(32))"
  2. 更新.env文件： 
    SESSION_SECRET=<生成的密钥>
  3. 重启服务： 
    ./fdas-ops.sh restart

A.4 服务启动异常

问题描述：

服务启动后健康检查失败，或容器反复重启。

应对措施：
  1. 查看容器日志： 
    docker logs fdas-app --tail=100
docker logs fdas-db --tail=100
  2. 检查端口冲突： 
    netstat -tlnp | grep 8000
netstat -tlnp | grep 5432
  3. 检查资源限制： 
    docker stats
df -h
free -m

A.5 数据迁移丢失

问题描述：

迁移后数据不完整或丢失。

应对措施：
  1. 验证备份完整性： 
    pg_restore -l backup.dump | head -20
  2. 恢复数据： 
    ./fdas-ops.sh restore
  3. 或执行回滚： 
    cd deployment-packages/migration
./rollback.sh

A.6 性能问题排查

问题描述：

API响应慢，数据库查询慢，前端加载慢。

应对措施：
  1. 检查数据库性能： 
    docker exec fdas-db psql -U fdas -d fdas -c "SELECT * FROM pg_stat_activity;"
docker exec fdas-db psql -U fdas -d fdas -c "SELECT * FROM pg_stat_database;"
  2. 检查慢查询： 
    # 启用慢查询日志（postgresql.conf）
log_min_duration_statement = 1000
  3. 优化索引： 
    # 分析查询
EXPLAIN ANALYZE SELECT * FROM forex_daily WHERE symbol_id = 'xxx';

# 创建索引
CREATE INDEX idx_symbol_date ON forex_daily(symbol_id, date DESC);
